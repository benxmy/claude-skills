# /morning-coffee — Daily Focus Picker

Arguments passed: `$ARGUMENTS`

## Steps

1. **Read the workboard** at `~/.claude/projects/memory/workboard.md`.
   Also read MEMORY.md for any specific reminders due today.

2. **Get today's date and pull calendar + emails + Webex** — run all in parallel:

   **Run these calls in parallel:**
   - `date '+%Y-%m-%d (%A)'`
   - `~/.config/claude-graph/bin/msgraph calendar 1` (today's meetings)
   - `~/.config/claude-graph/bin/msgraph email recent 2 --unread --max 20` (unread emails from last 48 hours)

   If msgraph commands fail (token expired, not configured), skip gracefully — these are additive, not required.

   **Webex triage** — Do NOT run the daily_summary.py script. Instead, read today's pre-computed triage file:
   ```
   ~/Projects/webex-agent/output/YYYY-MM-DD-triage.md
   ```
   (Use today's date.) This file is generated by the morning cron at 8:30am. If the file doesn't exist or is from a previous day, say "Webex triage not yet available" and continue without it.

   The triage contains prioritized groups: Blocked on You, Decisions Made Without You, Opportunities to Add Value, and FYI.

3. **Show the day at a glance** — before diving into the workboard, show what today looks like:

   **Calendar:**
   - List today's meetings in a compact table: time, duration, subject
   - Flag meetings that need prep (1:1s, external stakeholders, presentations)
   - Note free blocks > 1 hour — these are where focus work happens
   - If the day is meeting-heavy (>4 hours of meetings), call that out explicitly

   **Email:**
   - Scan unread emails for anything that needs action today:
     - Direct requests or questions addressed to Ben
     - Emails from leadership or skip-level (flag as high-priority context)
     - Meeting follow-ups or pre-reads for today's meetings
   - Summarize in 2-3 bullets max — don't list every email, just the ones that matter for today's decisions
   - If nothing notable, just say "Inbox is clean" and move on

   **Webex:**
   - Show the triage output from step 2b, organized by priority:
     - **Blocked on You** — people waiting for your response. These are the most urgent.
     - **Decisions Made Without You** — things decided that affect your work.
     - **Opportunities** — where your input could help proactively.
     - **FYI** — context only, no action needed.
   - Keep it compact — show the space name, who's involved, and what's needed. Don't dump the full triage verbatim; synthesize it into the morning briefing format.
   - If Webex triage returned nothing or failed, just say "Webex is quiet" and move on.

4. **Route Webex context to projects.** Before the status scan, check whether any Webex triage items relate to active workboard projects. For each match:
   - Read the project's memory file (e.g., `cii-data-in-duo.md`, `entra-lite-cii.md`, etc.)
   - Append a dated context note with the relevant Webex signal — who said what, what decision was made, what's blocked
   - Use `Edit` to append (never overwrite) under a `## Recent Context` section at the bottom of the file. Create the section if it doesn't exist.
   - Keep entries concise: one line per item, dated, with the space name and key info.
   - This ensures that when Ben later works on a project, the relevant Webex context is already there.

   Example format in a project memory file:
   ```
   ## Recent Context
   - 2026-05-04: [Duo Everywhere space] William Hansch shared updated Firewall slides — review before May 8 deadline
   - 2026-05-04: [DM - Simon] Still no response on tenant connection explanation — 5 days waiting
   ```

5. **Show a quick status scan** — go through every Active workboard item and flag:
   - **Stalled**: anything where the "next step" hasn't changed in 3+ days, or where a blocker has been sitting unresolved
   - **Waiting**: items blocked on other people — remind Ben to follow up
   - **Quick wins**: items where the next step is small and concrete (< 30 min)

6. **Ask Ben to pick 2-3 focus items for today.** Frame it as:
   > "Here's what's on your board. What do you want to focus on today?"

   Then list the active items grouped by category (Work, Plugins/Tools, Personal), showing just the name and next step for each.

   After listing, offer a suggestion based on:
   - What has external stakeholders waiting (highest priority)
   - What aligns with today's meetings (prep that would be useful before a call)
   - What's been stalled longest
   - What fits in the free blocks between meetings
   - What's a quick win that would feel good to clear

7. **Protect writing time.** Always ask:
   > "Do you want to block any time for writing today?"

   Don't make this optional or an afterthought — it's a core value, treat it like a real commitment. If the calendar has a good free block, suggest it specifically.

8. **Surface any due reminders** from MEMORY.md's Specific Reminders section (date matches today).

9. **Update the workboard** — set `Last reviewed: YYYY-MM-DD` at the top.

10. **Save the plan** — once Ben confirms his focus items, write them to
   `~/.claude/projects/memory/daily/YYYY-MM-DD.md` (using today's date) and also to `~/today-plan.md` (overwritten daily for easy access) so `/today` can
   recall it. Format:
   ```
   # Today's Plan — YYYY-MM-DD (Day)

   ## Meetings
   - HH:MM — Subject (note if prep needed)
   ...

   ## Focus Items
   1. **Item** — brief context
   2. **Item** — brief context
   ...

   ## Webex Triage
   - Blocked: ...
   - Decisions: ...
   - Opportunities: ...

   ## Email Follow-ups
   - Brief description of any emails needing action

   ## Context Routed to Projects
   - <project> — <what was added>
   ```

## Tone

Be direct and practical, like a good chief of staff. Don't be overly cheerful or add filler. Help Ben see the landscape clearly and make a decision. If the board is overloaded, say so plainly — "You have 15 active items. That's a lot. Want to park some of these?" If the day is packed with meetings, be honest — "You have 5 hours of meetings today. Realistically you'll get one focus block."

## After the plan is set

Once Ben confirms his focus items, close with:
> "Need to route something, check what's stalled, or get a cross-project status update? Use `/cos`."

## If `$ARGUMENTS` contains a message

Treat it as Ben telling you what he wants to focus on. Log it, update the workboard's last-reviewed date, and confirm the plan for the day. Still pull calendar and emails, still flag anything stalled or waiting.
